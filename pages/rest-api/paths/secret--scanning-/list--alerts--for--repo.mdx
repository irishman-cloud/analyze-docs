# GET - `/repos/{owner}/{repo}/secret-scanning/alerts`

List secret scanning alerts for a repository

## Parameters

| Name | Description | Type | Example |
| :---- | :---- | :---- | :---- |
| owner <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> | The account owner of the repository. The name is not case sensitive. | string in path | `undefined` |
| repo <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> | The name of the repository. The name is not case sensitive. | string in path | `undefined` |
| state  | Set to `open` or `resolved` to only list secret scanning alerts in a specific state. | string in query | `undefined` |
| secret_type  | A comma-separated list of secret types to return. By default all secret types are returned.
See "[Secret scanning patterns](https://docs.github.com/code-security/secret-scanning/secret-scanning-patterns#supported-secrets-for-advanced-security)"
for a complete list of secret types. | string in query | `undefined` |
| resolution  | A comma-separated list of resolutions. Only secret scanning alerts with one of these resolutions are listed. Valid resolutions are `false_positive`, `wont_fix`, `revoked`, `pattern_edited`, `pattern_deleted` or `used_in_tests`. | string in query | `undefined` |
| sort  | The property to sort the results by. `created` means when the alert was created. `updated` means when the alert was updated or resolved. | string in query | `undefined` |
| direction  | The direction to sort the results by. | string in query | `undefined` |
| page  | Page number of the results to fetch. | integer in query | `undefined` |
| per_page  | The number of results per page (max 100). | integer in query | `undefined` |
| before  | A cursor, as given in the [Link header](https://docs.github.com/rest/overview/resources-in-the-rest-api#link-header). If specified, the query only searches for events before this cursor. To receive an initial cursor on your first request, include an empty "before" query string. | string in query | `undefined` |
| after  | A cursor, as given in the [Link header](https://docs.github.com/rest/overview/resources-in-the-rest-api#link-header). If specified, the query only searches for events after this cursor.  To receive an initial cursor on your first request, include an empty "after" query string. | string in query | `undefined` |
