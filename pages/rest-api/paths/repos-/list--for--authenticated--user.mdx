# GET - `/user/repos`

List repositories for the authenticated user

## Parameters

| Name | Description | Type | Example |
| :---- | :---- | :---- | :---- |
| visibility  | Limit results to repositories with the specified visibility. | string in query | `undefined` |
| affiliation  | Comma-separated list of values. Can include:  
\* `owner`: Repositories that are owned by the authenticated user.  
\* `collaborator`: Repositories that the user has been added to as a collaborator.  
\* `organization_member`: Repositories that the user has access to through being a member of an organization. This includes every repository on every team that the user is on. | string in query | `undefined` |
| type  | Limit results to repositories of the specified type. Will cause a `422` error if used in the same request as **visibility** or **affiliation**. | string in query | `undefined` |
| sort  | The property to sort the results by. | string in query | `undefined` |
| direction  | The order to sort by. Default: `asc` when using `full_name`, otherwise `desc`. | string in query | `undefined` |
| per_page  | The number of results per page (max 100). | integer in query | `undefined` |
| page  | Page number of the results to fetch. | integer in query | `undefined` |
| since  | Only show notifications updated after the given time. This is a timestamp in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format: `YYYY-MM-DDTHH:MM:SSZ`. | string in query | `undefined` |
| before  | Only show notifications updated before the given time. This is a timestamp in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format: `YYYY-MM-DDTHH:MM:SSZ`. | string in query | `undefined` |
