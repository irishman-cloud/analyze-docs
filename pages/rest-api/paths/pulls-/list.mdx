# GET - `/repos/{owner}/{repo}/pulls`

List pull requests

## Parameters

| Name | Description | Type | Example |
| :---- | :---- | :---- | :---- |
| owner <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> | The account owner of the repository. The name is not case sensitive. | string in path | `undefined` |
| repo <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> | The name of the repository. The name is not case sensitive. | string in path | `undefined` |
| state  | Either `open`, `closed`, or `all` to filter by state. | string in query | `undefined` |
| head  | Filter pulls by head user or head organization and branch name in the format of `user:ref-name` or `organization:ref-name`. For example: `github:new-script-format` or `octocat:test-branch`. | string in query | `undefined` |
| base  | Filter pulls by base branch name. Example: `gh-pages`. | string in query | `undefined` |
| sort  | What to sort results by. `popularity` will sort by the number of comments. `long-running` will sort by date created and will limit the results to pull requests that have been open for more than a month and have had activity within the past month. | string in query | `undefined` |
| direction  | The direction of the sort. Default: `desc` when sort is `created` or sort is not specified, otherwise `asc`. | string in query | `undefined` |
| per_page  | The number of results per page (max 100). | integer in query | `undefined` |
| page  | Page number of the results to fetch. | integer in query | `undefined` |
