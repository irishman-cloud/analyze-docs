# GET - `/repos/{owner}/{repo}/assignees/{assignee}`

Check if a user can be assigned

## Parameters

| Name | Description | Type | Example |
| :---- | :---- | :---- | :---- |
| owner <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> | The account owner of the repository. The name is not case sensitive. | string in path | `undefined` |
| repo <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> | The name of the repository. The name is not case sensitive. | string in path | `undefined` |
| assignee <span class="bg-blue-100 text-blue-800 text-xs font-medium me-2 px-2.5 py-0.5 rounded dark:bg-blue-900 dark:text-blue-300">Required</span> |  | string in path | `undefined` |

## Responses

Blah blah blah.

<Tabs items={["204", "404"]}>

<Tabs.Tab>

If the `assignee` can be assigned to issues in the repository, a `204` header with no content is returned.

```json

{}

```

</Tabs.Tab>

<Tabs.Tab>

Otherwise a `404` status code is returned.

```json

{
    "message": "",
    "documentation_url": "",
    "url": "",
    "status": ""
}

```

</Tabs.Tab>
